# 정적 고객지원 웹사이트 디자인 스펙

- 날짜: 2026-04-19
- 범위: MVP 1차
- 목적: 기존 고객의 FAQ 자체 해결률을 높여 문의량을 줄인다.

## 1) 목표와 성공 기준

### 목표
기존 고객이 문의 전에 스스로 답을 찾을 수 있는 정적 웹사이트를 제공한다.

### 1차 성공 기준
- 고객 문의량 감소 (FAQ 기반 자체 해결 증가)

## 2) 대상 사용자
- 기존 고객

## 3) MVP 범위
- 페이지 수: 2~3페이지
  - 홈
  - FAQ
  - 문의

## 4) 정보 구조(IA)

### 홈
- 최상단: 공지/업데이트 배너(단일 현재 공지 슬롯)
- 주요 콘텐츠: 핵심 안내, 고객 후기 하이라이트
- 주요 행동 유도: FAQ 이동 CTA

### FAQ
- 탐색 방식: 카테고리 중심
- 카테고리:
  1. 시작하기
  2. 요금
  3. 정책
  4. 문제해결

### 문의
- 1순위 채널: 채팅 링크
- 보조 정보: 최소 연락 정보(필요 시)

## 5) 기술 아키텍처
- 구현: 순수 정적 HTML/CSS, 필요 최소 JavaScript
- 배포 형태: 정적 파일 배포
- 데이터 저장소/백엔드: 없음 (MVP 범위 외)

## 6) 콘텐츠 운영 방식
- 운영 주체: 운영팀
- 업데이트 주기: 필요 시 수시
- 운영 플로우:
  1. 운영팀이 수정 요청
  2. 개발자가 콘텐츠 텍스트 수정
  3. 재배포

### 운영 원칙
- 공지는 홈 상단 배너에서 단일 최신 공지로 노출
- FAQ는 반복 문의 항목을 카테고리별로 누적 추가
- 문의량 변화를 기준으로 FAQ 보강 우선순위를 조정

## 7) 에러/품질 전략
- 런타임 복잡 예외 처리보다 사전 검수 중심
- 사전 검수 항목:
  - 깨진 링크
  - 오탈자
  - 잘못된 앵커/내부 이동

## 8) 접근성/사용성 기준
- 모바일 우선 반응형
- 핵심 경로(공지 → FAQ → 문의) 3클릭 이내
- 명확한 제목 계층
- 충분한 색 대비
- 키보드 포커스 확인

## 9) 테스트 전략

### 골든 패스 수동 점검
1. 홈에서 최신 공지 확인
2. FAQ로 이동
3. 카테고리 탐색으로 답변 탐색
4. 필요 시 문의 페이지에서 채팅 링크 진입

### 회귀 점검
- 전 페이지 링크 유효성 점검
- 모바일 화면 레이아웃 점검

## 10) 비범위(Out of Scope)
- CMS 도입
- 검색 기반 FAQ 엔진
- 사용자 계정 기능
- 동적 백엔드/API 연동

## 11) 향후 확장 방향
- MVP 안정화 후 헤드리스 CMS 도입 검토
- 운영팀 직접 수정 가능한 구조로 전환 검토
